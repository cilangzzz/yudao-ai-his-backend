# HIS 模块实现工作流

> 自动化实现 HIS 系统核心业务模块（M01门诊/M02住院/M06药品）
> **注意**: M09 系统管理由 yudao-module-system 自带，无需重复开发

## 快速启动

### 完整执行（所有模块）

```
/Workflow his-implementation
```

### 执行单个 Sprint

```
/Workflow his-implementation {"sprint": "sprint1"}
```

### 执行单个子模块

```
/Workflow his-implementation {"submodule": "M01-01"}
```

## 模块依赖说明

### 系统自带模块（无需开发）

| 模块 | 来源 | 功能 |

| 模块 | 子模块数 | 核心功能 |

## 文档生成工作流

### 菜单文档生成

生成前端开发人员可阅读的菜单配置文档。

```
/Workflow his-menu-doc-generator
```

**参数选项**:
- 完整执行: 生成所有 HIS 模块的菜单文档
- 单模块执行: `/Workflow his-menu-doc-generator {"module": "M01"}`

**输出**:
- `docs/his/{模块编号}-{模块名称}/{模块编号}-菜单配置文档.md`

**文档内容**:
- 菜单层级结构树
- 页面菜单配置表（路由路径、组件路径、组件名称、图标）
- 按钮权限配置表
- 前端路由配置示例
- Vue 组件目录结构建议

## 文档同步工作流

### 同步需求、设计、数据文档

从外部设计文档目录同步到项目 docs/his 目录。

```
/his-doc-sync
```

**参数选项**:
- 完整同步: 同步所有类型文档
- 同步指定模块: `/his-doc-sync {"module": "M01"}`
- 同步指定类型: `/his-doc-sync {"type": "需求"}`

**文档类型**:
- 需求文档: 需求开发目录
- 设计文档: 系统设计目录
- 数据文档: 概要设计目录

## 实现核查工作流

### 核查代码与设计文档一致性

验证代码实现是否符合设计文档要求。

```
/his-implementation-check
```

**参数选项**:
- 完整核查: 检查所有模块
- 核查指定模块: `/his-implementation-check {"module": "M01"}`
- 核查指定类型: `/his-implementation-check {"checkType": "database"}`

**检查类型**:
| 类型 | 说明 |
|------|------|
| database | 数据库表结构与设计文档对比 |
| entity | DO 实体类与数据库表对应 |
| api | Controller 接口与设计文档对比 |
| business | 业务逻辑与功能需求对比 |

**输出**: `docs/his/IMPLEMENTATION_CHECK_REPORT.md`

## 项目修复工作流

### Maven 编译检测与自动修复

检测并修复项目编译问题。

```
/his-project-fix
```

**参数选项**:
- 完整修复: 自动迭代修复直到编译成功
- 仅编译检测: `/his-project-fix {"mode": "check"}`
- 指定模块修复: `/his-project-fix {"module": "his"}`

**修复类型**:
| 问题类型 | 自动修复 |
|----------|----------|
| 编译错误 | ✅ |
| 依赖问题 | ✅ |
| 语法错误 | ✅ |
| 导入错误 | ✅ |
| 注解问题 | ✅ |
| 命名规范 | ❌ (仅提示) |
