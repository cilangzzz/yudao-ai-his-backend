# HIS 模块实现工作流

> 自动化实现 HIS 系统核心业务模块（M01门诊/M02住院/M06药品）
> **注意**: M09 系统管理由 yudao-module-system 自带，无需重复开发

## 快速启动

### 完整执行（所有模块）

```
/Workflow his-implementation
```

### 执行单个 Sprint

```
/Workflow his-implementation {"sprint": "sprint1"}
```

### 执行单个子模块

```
/Workflow his-implementation {"submodule": "M01-01"}
```

| sprint3 | 第5-6周 | M01 门诊 | M01-03 收费管理 | - |

| sprint5 | 第9-11周 | M02 住院 | M02-01 入院管理 | - |
| sprint6 | 第12-14周 | M02 住院 | M02-02 医生工作站 | - |

## 里程碑验收

| 里程碑 | 验收标准 |
|--------|----------|
| M1 门诊闭环 | 挂号→接诊→收费→发药流程贯通 |
| M2 住院闭环 | 入院→医嘱→护理→出院闭环，eMAR核对率100% |
| M3 药品管理 | 药库入库→出库→处方审核→特殊药品管理可用 |

## 模块依赖说明

### 系统自带模块（无需开发）

| 模块 | 来源 | 功能 |
|------|------|------|
| **M09 系统管理** | yudao-module-system | 用户、角色、权限、部门、字典、日志 |

### 需要开发的 HIS 模块

| 模块 | 子模块数 | 核心功能 |
|------|----------|----------|
| **M01 门诊管理** | 4 | 挂号、医生工作站、收费、药房 |
| **M02 住院管理** | 5 | 入院、医生工作站、护理、床位、出院 |
| **M06 药品管理** | 4 | 药库、采购、处方审核、特殊药品 |

## 输出物

每个子模块生成：

- DO 实体类
- Mapper 接口
- VO 类（SaveReqVO、PageReqVO、RespVO）
- Service 接口和实现
- Controller
- 错误码枚举
- 单元测试
