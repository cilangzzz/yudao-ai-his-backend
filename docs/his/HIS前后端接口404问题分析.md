## 二、404 问题分类

### 类型 A：后端 Controller 完全缺失

| 前端 API 文件 | 调用路径 | 缺失的 Controller | 数据库表 |
|--------------|---------|------------------|---------|
| `api/his/basic/department/index.ts` | `/his/basic/department/*` | HisDepartmentController | `his_department` |
| `api/his/basic/staff/index.ts` | `/his/basic/staff/*` | HisStaffController | `his_staff` |
| `api/his/basic/chargeItem/index.ts` | `/his/basic/charge-item/*` | HisChargeItemController | `his_charge_item` |

**原因**: 菜单数据定义了这些功能模块，前端组件和 API 已开发，但后端 Controller 未实现。

**影响**: 打开「基础数据」下的「科室管理」「医护人员」「收费项目」菜单，所有接口都会 404。

---

### 类型 B：路径前缀不匹配

| 前端 API 文件 | 前端调用路径 | 后端实际路径 | 后端 Controller |
|--------------|------------|-------------|----------------|
| `api/his/basic/ward/index.ts` | `/his/basic/ward/*` | `/his/ward/*` | HisWardController |

**原因**: 前端 API 文件按照菜单结构组织（放在 `basic` 子目录），但后端 Controller 没有使用 `/his/basic` 路径前缀。

**影响**: 打开「基础数据」下的「病区管理」菜单，所有接口都会 404。

---

### 类型 C：接口名称不匹配

| 模块 | 前端接口 | 后端接口 | 问题 |
|------|---------|---------|------|
| 床位管理 | `/his/bed/available` | `/his/bed/empty-beds` | 接口名不同 |
| 床位管理 | `/his/bed/assign` | `/his/bed/occupy` | 接口名不同 |
| 床位管理 | `/his/bed/overview` | 无 | 后端缺少统计接口 |
| 床位管理 | `/his/bed/transfer` | 无 | 后端缺少转床接口 |

**原因**: 前端和后端命名习惯不同（如 `available` vs `empty-beds`），或后端缺少某些查询接口。

---

## 三、解决方案

### 方案 1：补充缺失的后端 Controller（推荐）

需要新增以下 3 个 Controller：

| Controller | 路径 | 对应前端 | 对应表 |
|-----------|------|---------|-------|
| HisDepartmentController | `/his/basic/department` | `api/his/basic/department/index.ts` | `his_department` |
| HisStaffController | `/his/basic/staff` | `api/his/basic/staff/index.ts` | `his_staff` |
| HisChargeItemController | `/his/basic/charge-item` | `api/his/basic/chargeItem/index.ts` | `his_charge_item` |

同时需要：
1. 将 HisWardController 的路径从 `/his/ward` 改为 `/his/basic/ward`
2. 补充床位管理的 `overview` 和 `transfer` 接口

---

### 方案 2：修改前端 API 调用路径

将前端 API 文件的调用路径改为匹配后端：

| 文件 | 修改内容 |
|------|---------|
| `api/his/basic/ward/index.ts` | 将 `/his/basic/ward` 改为 `/his/ward` |
| `api/his/inpatient/bed/index.ts` | 将 `/his/bed/available` 改为 `/his/bed/empty-beds` |
| `api/his/inpatient/bed/index.ts` | 将 `/his/bed/assign` 改为 `/his/bed/occupy` |

但这种方式会导致前端 API 文件结构与菜单结构不一致，不建议。
