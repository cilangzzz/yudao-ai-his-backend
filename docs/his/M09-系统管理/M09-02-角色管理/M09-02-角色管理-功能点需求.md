## 1. 功能概述

### 1.1 功能定位

角色管理是系统管理子系统的权限配置模块，提供角色定义、权限分配、角色继承等功能。系统基于RBAC（基于角色的访问控制）模型，实现菜单权限、按钮权限、数据权限的灵活配置。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 权限分离原则 | 关键角色权限分离率100% |
| 效率目标 | 权限配置效率 | 角色配置时间≤5分钟 |
| 规范目标 | 权限审计 | 权限变更审计覆盖率100% |
| 灵活目标 | 权限灵活性 | 支持多级权限继承 |

### 1.3 功能范围

## 4. 接口设计

### 4.1 角色创建接口

**接口路径**: `POST /api/sys/role`

**请求参数**:
```json
{
  "roleName": "门诊医生",
  "roleCode": "op_doctor",
  "roleDesc": "门诊医生角色",
  "menuIds": [1, 2, 3],
  "dataScope": 3
}
```

### 4.2 角色权限配置接口

**接口路径**: `POST /api/sys/role/{roleId}/permission`

**请求参数**:
```json
{
  "menuIds": [1, 2, 3, 5, 6],
  "buttonIds": [1, 2, 3],
  "dataScope": 3
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-SYS-006 | 角色唯一 | 角色标识必须唯一 |
| BR-SYS-007 | 权限分离 | 关键角色权限必须分离 |
| BR-SYS-008 | 角色继承 | 支持角色继承，子角色权限不能超过父角色 |
