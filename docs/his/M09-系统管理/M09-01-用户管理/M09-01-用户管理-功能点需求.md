## 1. 功能概述

### 1.1 功能定位

用户管理是系统管理子系统的基础模块，提供用户账号生命周期管理、密码管理、用户状态管理等功能。系统基于RBAC（基于角色的访问控制）模型实现细粒度的权限控制，支持菜单权限、按钮权限、数据权限三级管理。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 账号安全管理 | 密码复杂度检查覆盖率100% |
| 效率目标 | 用户管理效率 | 用户创建时间≤2分钟 |
| 规范目标 | 权限分配规范 | 权限审计覆盖率100% |
| 合规目标 | 等保三级合规 | 登录日志保留≥3年 |

### 1.3 功能范围

```

│   ├── 用户创建
```

## 4. 接口设计

### 4.1 用户创建接口

**接口路径**: `POST /api/sys/user`

**请求参数**:
```json
{
  "username": "doctor03",
  "name": "王医生",
  "empNo": "EMP20260601",
  "gender": "M",
  "phone": "13800000000",
  "email": "wang@hospital.com",
  "idCard": "330102****1234",
  "deptId": 10,
  "wardId": 101,
  "postId": 1,
  "titleId": 3,
  "roleIds": [1, 2],
  "dataScope": 2,
  "password": "Abc@123456",
  "forceChangePassword": true
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "创建成功",
  "data": {
    "userId": 1001,
    "username": "doctor03"
  }
}
```

### 4.2 用户查询接口

**接口路径**: `GET /api/sys/user/list`

**请求参数**:
```
username=&name=&status=1&deptId=10&roleId=1&pageNum=1&pageSize=20
```

**响应结果**:
```json
{
  "code": 200,
  "data": {
    "total": 156,
    "list": [
      {
        "userId": 1001,
        "username": "doctor01",
        "name": "张医生",
        "deptName": "内科",
        "roles": ["门诊医生"],
        "status": 1,
        "createTime": "2026-06-01 10:00:00"
      }
    ]
  }
}
```

### 4.3 密码重置接口

**接口路径**: `POST /api/sys/user/{userId}/reset-password`

**请求参数**:
```json
{
  "newPassword": "Xyz@654321"
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "密码重置成功"
}
```

### 4.4 用户锁定/解锁接口

**接口路径**: `POST /api/sys/user/{userId}/lock`

**请求参数**:
```json
{
  "lockReason": "多次登录失败"
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "用户已锁定"
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-SYS-001 | 用户名唯一 | 用户账号必须唯一，支持用户名/工号登录 |
| BR-SYS-002 | 密码复杂度 | 密码必须满足复杂度要求（8位以上，含大小写字母+数字+特殊字符） |
| BR-SYS-003 | 登录失败锁定 | 连续登录失败5次锁定账户30分钟 |
| BR-SYS-004 | 敏感操作审计 | 敏感操作必须二次确认并记录审计日志 |
| BR-SYS-005 | 数据权限控制 | 用户只能访问其数据权限范围内的数据 |
