## 1. 病历书写与质控流程

```mermaid
flowchart TD
    A([患者入院]) --> B[入院记录-24h内完成]
    B --> C[首次病程记录-8h内完成]
    C --> D[日常病程记录]
    D --> E[上级医师查房记录]
    E --> F[系统自动质控检查]
    F --> G{质控结果}
    G -- 时限超期 --> H[发送质控提醒]
    H --> I[医生补写病历]
    G -- 内容缺失 --> J[标记缺失项]
    J --> K[医生补充完善]
    G -- 通过 --> L[病历保存]
    I --> L
    K --> L
    L --> M[上级医师审签]
    M --> N{审签结果}
    N -- 通过 --> O[病历归档]
    N -- 退回 --> P[修改意见]
    P --> Q[医生修改]
    Q --> M
    O --> R[归档锁定]
    R --> S([病历完成])
```

---

## 2. 病历封存流程（医疗纠纷场景）

```mermaid
flowchart TD
    A([医疗纠纷发生]) --> B[医务科发起封存申请]
    B --> C[系统锁定病历]
    C --> D[生成病历副本]
    D --> E[患方确认封存]
    E --> F[双方签字]
    F --> G[封存病历保管]
    G --> H[记录封存日志]
    H --> I([封存完成])
```
