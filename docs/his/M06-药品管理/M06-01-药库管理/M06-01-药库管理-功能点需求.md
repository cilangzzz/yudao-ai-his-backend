## 1. 功能概述

### 1.1 功能定位

药库管理是药品管理子系统的核心模块，负责药品入库验收、库存管理、出库配送、盘点管理、效期管理、批号追踪等功能。系统实现药品全生命周期管理，保障药品供应链的安全和效率。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 安全目标 | 药品质量安全 | 入库验收合格率100% |
| 准确目标 | 库存数据准确 | 库存盘点误差率≤0.5% |
| 效率目标 | 药品周转效率 | 效期预警及时率100% |
| 合规目标 | 特殊药品合规 | 麻醉药品五专管理合规率100% |

## 4. 接口设计

### 4.1 入库验收接口

**接口路径**: `POST /api/pharm/inbound/verify`

**请求参数**:
```json
{
  "inboundId": 10001,
  "verifyResult": true,
  "verifyItems": [
    {"drugId": 1001, "actualQty": 500, "batchNo": "20260601", "expireDate": "2028-06-01"}
  ],
  "verifyNotes": "质检合格"
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "入库验收成功",
  "data": {
    "inboundId": 10001,
    "verifyTime": "2026-06-17 10:00:00"
  }
}
```

### 4.2 库存查询接口

**接口路径**: `GET /api/pharm/stock/list`

**响应结果**:
```json
{
  "code": 200,
  "data": [
    {
      "drugId": 1001,
      "drugName": "阿莫西林胶囊",
      "spec": "0.5g×24",
      "stockQty": 500,
      "batchNo": "20260601",
      "expireDate": "2028-06-01",
      "expireStatus": "normal"
    }
  ]
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-PHARM-001 | 入库验收 | 药品入库必须验收（品名、规格、批号、效期、数量） |
| BR-PHARM-002 | 效期预警 | 近效期药品（≤3个月）自动预警 |
| BR-PHARM-003 | 出库先进先出 | 药品出库遵循"先进先出"原则 |
| BR-PHARM-004 | 批号追踪 | 药品批号全程可追踪 |
