# 给药管理 API 接口文档

> **模块**: 护理工作站
> **Controller**: `HisMedicationAdminController`
> **基础路径**: `/his/medication-admin`
> **版本**: v1.0
> **更新日期**: 2026-06-18

---

## 接口概览

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 创建给药记录 | POST | `/create` | `his:medication-admin:create` | 创建给药记录 |
| 更新给药记录 | PUT | `/update` | `his:medication-admin:update` | 更新给药信息 |
| 删除给药记录 | DELETE | `/delete` | `his:medication-admin:delete` | 删除给药记录 |
| 获取给药记录 | GET | `/get` | `his:medication-admin:query` | 根据ID查询 |
| 获取给药分页 | GET | `/page` | `his:medication-admin:query` | 分页查询给药记录 |
| 住院给药记录 | GET | `/list-by-admission` | `his:medication-admin:query` | 查询住院给药列表 |
| 待执行给药列表 | GET | `/pending-list` | `his:medication-admin:query` | 查询待执行给药 |
| 腕带扫描验证 | POST | `/verify-wristband` | `his:medication-admin:execute` | 闭环给药腕带验证 |
| 药品条码验证 | POST | `/verify-drug` | `his:medication-admin:execute` | 闭环给药药品验证 |
| 给药执行确认 | POST | `/confirm` | `his:medication-admin:execute` | 闭环给药确认 |
| 记录未执行原因 | POST | `/record-not-executed` | `his:medication-admin:execute` | 记录未执行原因 |

### 2. 腕带扫描验证（闭环给药）

**请求**
- 方法: `POST`
- 路径: `/his/medication-admin/verify-wristband`
- 权限: `his:medication-admin:execute`
- Content-Type: `application/json`

**请求体** (`WristbandVerifyReqVO`)

```json
{
  "adminId": 1,
  "wristbandCode": "WB20240001",
  "nurseId": 200
}
```

**响应**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "success": true,
    "patientId": 1,
    "patientName": "张三",
    "inpatientNo": "ZY20240001",
    "message": "腕带验证通过"
  }
}
```

### 4. 给药执行确认（闭环给药）

**请求**
- 方法: `POST`
- 路径: `/his/medication-admin/confirm`
- 权限: `his:medication-admin:execute`
- Content-Type: `application/json`

**请求体** (`MedicationAdminConfirmReqVO`)

```json
{
  "adminId": 1,
  "nurseId": 200,
  "nurseName": "张护士",
  "wristbandVerified": true,
  "drugBarcodeVerified": true,
  "executeTime": "2024-06-18 14:00:00",
  "remark": ""
}
```

**响应**

```json
{
  "code": 0,
  "msg": "success",
  "data": 1  // 给药记录ID
}
```

### 5. 记录未执行原因

**请求**
- 方法: `POST`
- 路径: `/his/medication-admin/record-not-executed`
- 权限: `his:medication-admin:execute`

**参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| adminId | Long | 是 | 给药记录ID |
| reason | String | 是 | 未执行原因 |
| reasonType | String | 是 | 原因类型 |

**响应**

```json
{
  "code": 0,
  "msg": "success",
  "data": true
}
```

## 给药状态

| 状态值 | 说明 |
|----|------|
| 0 | 待执行 |
| 1 | 已执行 |
| 2 | 未执行 |
| 3 | 已取消 |

---

## 未执行原因类型

| 值 | 说明 |
|----|------|
| PATIENT_REFUSED | 患者拒绝 |
| ADVERSE_REACTION | 不良反应 |
| DRUG_UNAVAILABLE | 药品不可用 |
| ORDER_CHANGED | 医嘱变更 |
| OTHER | 其他原因 |

---

## 错误码

| 错误码 | 说明 |
|--------|------|
| 1_016_005_001 | 给药记录不存在 |
| 1_016_005_002 | 给药已执行 |
| 1_016_005_003 | 药品条码不匹配 |
| 1_016_005_004 | 患者腕带不匹配 |

## 前端对接示例

### 获取待执行给药列表

```javascript
const getPendingMedicationList = async (admissionId) => {
  const res = await request({
    url: '/his/medication-admin/pending-list',
    method: 'GET',
    params: { admissionId }
  })
  return res.data
}
```

### 闭环给药验证与执行

```javascript
// 1. 腕带扫描验证
const verifyWristband = async (adminId, wristbandCode, nurseId) => {
  const res = await request({
    url: '/his/medication-admin/verify-wristband',
    method: 'POST',
    data: { adminId, wristbandCode, nurseId }
  })
  return res.data
}

// 2. 药品条码验证
const verifyDrugBarcode = async (adminId, drugBarcode, nurseId) => {
  const res = await request({
    url: '/his/medication-admin/verify-drug',
    method: 'POST',
    data: { adminId, drugBarcode, nurseId }
  })
  return res.data
}

// 3. 确认执行
const confirmMedication = async (data) => {
  const res = await request({
    url: '/his/medication-admin/confirm',
    method: 'POST',
    data
  })
  return res.data
}
```
