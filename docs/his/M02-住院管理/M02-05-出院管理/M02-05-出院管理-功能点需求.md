## 1. 功能概述

### 1.1 功能定位

出院管理是住院管理子系统的出院流程模块，提供出院申请、出院结算、出院带药、出院指导、病案归档等功能。系统实现出院流程的规范化管理，确保出院结算准确、病案归档及时。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短出院办理时间 | 平均出院办理时间≤30分钟 |
| 准确目标 | 结算准确率 | 出院结算差错率≤0.01% |
| 规范目标 | 病案归档及时 | 出院后3天内归档率100% |
| 服务目标 | 出院指导覆盖 | 出院指导覆盖率100% |

## 4. 接口设计

### 4.1 出院申请接口

**接口路径**: `POST /api/ip/discharge/apply`

**请求参数**:
```json
{
  "admissionId": 10001,
  "dischargeDate": "2026-06-25",
  "dischargeType": 1,
  "dischargeDiagnosis": [
    {"diagnosisCode": "J18.9", "diagnosisStatus": 1}
  ]
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "出院申请已提交",
  "data": {
    "dischargeId": 50001,
    "status": "pending_approval"
  }
}
```

### 4.2 出院结算接口

**接口路径**: `POST /api/ip/discharge/settlement`

**请求参数**:
```json
{
  "dischargeId": 50001
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "出院结算成功",
  "data": {
    "settlementId": 60001,
    "totalAmount": 4400.00,
    "insurancePay": 2500.00,
    "personalPay": 1900.00,
    "refundAmount": 100.00
  }
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-IP-014 | 出院前医嘱停止 | 出院前必须停止所有长期医嘱 |
| BR-IP-015 | 出院结算审核 | 出院结算必须完成费用审核 |
| BR-IP-016 | 医保出院结算 | 医保患者出院必须完成医保结算 |
| BR-IP-017 | 病案归档锁定 | 病案归档后不可修改 |
