## 1. 功能概述

### 1.1 功能定位

入院管理是住院管理子系统的入口模块，负责患者入院登记、床位分配、预交金管理、医保登记、入院评估等功能。系统支持门诊转住院、急诊转住院等多种入院方式，实现住院流程的规范化管理。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短入院办理时间 | 平均入院登记时间≤10分钟 |
| 准确目标 | 床位分配准确 | 床位分配准确率100% |
| 安全目标 | 入院评估及时完成 | 入院评估4小时内完成率100% |
| 管理目标 | 预交金及时收取 | 入院预交金收取率100% |

## 4. 接口设计

### 4.1 入院登记接口

**接口路径**: `POST /api/ip/admission`

**请求参数**:
```json
{
  "patientId": 1001,
  "registerId": 10001,
  "admissionType": 1,
  "deptId": 20,
  "wardId": 201,
  "bedId": 102,
  "diagnosisCode": "J18.9",
  "conditionLevel": 2,
  "depositAmount": 5000.00,
  "payType": 3
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "入院登记成功",
  "data": {
    "admissionId": 10001,
    "admissionNo": "ZY202606170001",
    "bedNo": "102",
    "depositBalance": 5000.00
  }
}
```

### 4.2 床位查询接口

**接口路径**: `GET /api/ip/bed/list`

**请求参数**:
```
wardId=201&status=1
```

**响应结果**:
```json
{
  "code": 200,
  "data": [
    {
      "bedId": 102,
      "bedNo": "102",
      "status": 1,
      "patientName": null,
      "admissionId": null
    },
    {
      "bedId": 104,
      "bedNo": "104",
      "status": 1,
      "patientName": null,
      "admissionId": null
    }
  ]
}
```

### 4.3 入院评估接口

**接口路径**: `POST /api/ip/admission/{admissionId}/assessment`

**请求参数**:
```json
{
  "fallRiskScore": 2,
  "fallRiskLevel": "中度风险",
  "fallPreventionMeasures": ["使用床栏", "呼叫器置于可及处"],
  "pressureRiskScore": 18,
  "pressureRiskLevel": "低风险",
  "educationItems": ["病区环境介绍", "探视制度说明", "住院须知讲解"]
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-IP-001 | 入院诊断关联 | 入院必须关联门诊诊断或急诊诊断 |
| BR-IP-002 | 预交金不足提醒 | 预交金不足时自动提醒，欠费超过阈值限制记账 |
| BR-IP-003 | 床位分配规则 | 床位分配需考虑性别、病种、感染等因素 |
| BR-IP-004 | 入院评估时限 | 入院护理评估（跌倒、压疮风险）必须在入院4小时内完成 |
| BR-IP-005 | 医保登记时限 | 医保患者入院后24小时内必须完成医保登记 |
