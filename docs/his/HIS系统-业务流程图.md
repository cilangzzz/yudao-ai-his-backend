## 1. 文档说明

### 1.1 编写目的

本文档描述YUDAO-AI-HIS系统的核心业务流程，采用文字描述结合Mermaid流程图语法的方式，清晰展示各业务场景的流程步骤、角色参与、数据流转和异常处理。所有流程基于真实医院业务场景设计，参考《全国医院信息化建设标准与规范》中的业务应用要求。

### 1.2 流程图符号说明

| 符号 | 含义 |
|------|------|
| [矩形] | 处理步骤/操作 |
| {菱形} | 判断/决策 |
| (圆角矩形) | 开始/结束 |
| [[子流程]] | 调用子流程 |
| //平行四边形// | 数据输入/输出 |

## 2. 门诊业务流程

### 2.1 门诊就诊全流程

**流程说明**：患者从挂号到取药完成一次完整的门诊就诊过程。

```
流程概览：
患者到达 → 挂号/签到 → 候诊 → 就诊 → 缴费 → 取药/检查 → 离院
```

#### 2.1.1 门诊就诊主流程

```mermaid
flowchart TD
    A([患者到达医院]) --> B{是否已预约?}
    B -- 是 --> C[预约签到]
    B -- 否 --> D[现场挂号]
    D --> E[选择科室/医生]
    E --> F[缴纳挂号费]
    F --> G[分配排队号]
    C --> G
    G --> H[进入候诊队列]
    H --> I[叫号就诊]
    I --> J[医生接诊]
    J --> K[书写门诊病历]
    K --> L[开具诊断]
    L --> M{是否需要检查检验?}
    M -- 是 --> N[开具检查/检验申请]
    N --> O[患者缴费]
    O --> P[执行检查/检验]
    P --> Q[获取结果]
    Q --> R[医生查看结果]
    M -- 否 --> R
    R --> S{是否需要开药?}
    S -- 是 --> T[开具处方]
    T --> U[处方审核-CDS]
    U --> V[患者缴费]
    V --> W[药房取药]
    S -- 否 --> X
    W --> X[就诊结束]
    X --> Y([患者离院])
```

#### 2.1.2 挂号流程详细

```mermaid
flowchart TD
    A([开始]) --> B{挂号方式}
    B -- 线上预约 --> C[选择就诊日期]
    C --> D[选择科室]
    D --> E[选择医生/时段]
    E --> F[确认预约信息]
    F --> G[在线支付挂号费]
    G --> H[生成预约挂号单]
    H --> I([预约成功])

    B -- 现场挂号 --> J[出示身份证/医保卡]
    J --> K[读取患者信息]
    K --> L{是否已建档?}
    L -- 否 --> M[新建患者档案]
    M --> N[录入基本信息]
    N --> O[分配患者ID]
    L -- 是 --> O
    O --> P[选择科室]
    P --> Q[选择挂号类型]
    Q --> R[选择医生]
    R --> S[计算挂号费]
    S --> T{是否医保?}
    T -- 是 --> U[医保身份验证]
    U --> V[医保挂号费计算]
    T -- 否 --> V
    V --> W[收取挂号费]
    W --> X[打印挂号单]
    X --> Y[分配排队号]
    Y --> Z([挂号完成])

    B -- 自助机挂号 --> AA[刷卡/扫码]
    AA --> AB[选择科室医生]
    AB --> AC[自助缴费]
    AC --> AD[打印挂号单]
    AD --> Z
```

#### 2.1.3 门诊收费流程

```mermaid
flowchart TD
    A([收费开始]) --> B[获取待收费项目]
    B --> C[汇总费用明细]
    C --> D[费用分类统计]
    D --> E{是否医保患者?}
    E -- 是 --> F[医保费用预结算]
    F --> G[计算医保报销金额]
    G --> H[计算个人自付金额]
    E -- 否 --> H
    H --> I[显示收费清单]
    I --> J[患者确认]
    J --> K{支付方式}
    K -- 现金 --> L[现金收款]
    K -- 医保 --> M[医保结算]
    K -- 微信/支付宝 --> N[扫码支付]
    K -- 银行卡 --> O[POS刷卡]
    L --> P[打印发票]
    M --> P
    N --> P
    O --> P
    P --> Q[更新收费状态]
    Q --> R([收费完成])
```

## 3. 住院业务流程

### 3.1 住院就诊全流程

**流程说明**：患者从入院登记到出院结算的完整住院过程。

```
流程概览：
入院登记 → 入科 → 医嘱开立 → 检查检验 → 治疗护理 → 手术(如需) → 出院申请 → 出院结算
```

#### 3.1.1 住院主流程

```mermaid
flowchart TD
    A([医生开具入院证]) --> B[患者办理入院]
    B --> C[入院登记]
    C --> D[缴纳预交金]
    D --> E[医保登记]
    E --> F[分配床位]
    F --> G[入科确认]
    G --> H[入院护理评估]
    H --> I[医生开立医嘱]
    I --> J[护士审核医嘱]
    J --> K{医嘱类型}
    K -- 药品医嘱 --> L[药房配药]
    L --> M[护士给药执行]
    K -- 检验医嘱 --> N[标本采集]
    N --> O[检验科检验]
    O --> P[结果报告]
    K -- 检查医嘱 --> Q[预约检查]
    Q --> R[执行检查]
    R --> S[检查报告]
    K -- 护理医嘱 --> T[护理执行]
    K -- 手术医嘱 --> U[手术流程]
    M --> V[费用自动记账]
    P --> V
    S --> V
    T --> V
    U --> V
    V --> W{是否需要出院?}
    W -- 否 --> I
    W -- 是 --> X[出院申请]
    X --> Y[出院审批]
    Y --> Z[出院带药]
    Z --> AA[出院结算]
    AA --> AB[医保结算]
    AB --> AC[打印出院小结]
    AC --> AD[释放床位]
    AD --> AE([患者出院])
```

#### 3.1.2 医嘱处理流程

```mermaid
flowchart TD
    A([医生开立医嘱]) --> B[选择医嘱类型]
    B --> C{医嘱类别}
    C -- 药品医嘱 --> D[选择药品]
    D --> E[设置用法用量]
    E --> F[CDS自动校验]
    F --> G{校验结果}
    G -- 有警告 --> H[显示警告信息]
    H --> I{医生决定}
    I -- 继续 --> J[确认开立]
    I -- 修改 --> D
    G -- 通过 --> J
    C -- 检验医嘱 --> K[选择检验项目]
    K --> J
    C -- 检查医嘱 --> L[选择检查项目]
    L --> J
    C -- 护理医嘱 --> M[选择护理项目]
    M --> J
    J --> N[医嘱提交]
    N --> O{长期/临时?}
    O -- 长期医嘱 --> P[长期医嘱生效]
    O -- 临时医嘱 --> Q[临时医嘱立即执行]
    P --> R[护士站接收]
    Q --> R
    R --> S[护士审核]
    S --> T{审核结果}
    T -- 通过 --> U[医嘱执行]
    T -- 退回 --> V[退回原因]
    V --> W[医生修改]
    W --> N
    U --> X[执行记录]
    X --> Y([医嘱完成])
```

#### 3.1.3 手术流程

```mermaid
flowchart TD
    A([手术申请]) --> B[术前检查]
    B --> C[术前评估]
    C --> D{评估结果}
    D -- 可手术 --> E[手术排期]
    D -- 暂缓 --> F[术前准备]
    F --> B
    E --> G[手术室安排]
    G --> H[术前访视]
    H --> I[知情同意签署]
    I --> J[术前准备]
    J --> K[患者接入手术室]
    K --> L[手术安全核查]
    L --> M[麻醉诱导]
    M --> N[手术实施]
    N --> O[术中记录]
    O --> P[手术结束]
    P --> Q[麻醉恢复]
    Q --> R[术后交接]
    R --> S[术后医嘱]
    S --> T[术后护理]
    T --> U[术后访视]
    U --> V([手术流程结束])
```

#### 3.1.4 出院结算流程

```mermaid
flowchart TD
    A([出院申请]) --> B[医生审核]
    B --> C[护士站确认]
    C --> D[停止长期医嘱]
    D --> E[出院带药开立]
    E --> F[费用汇总]
    F --> G[费用审核]
    G --> H{是否有未结费用?}
    H -- 是 --> I[补充计费]
    I --> F
    H -- 否 --> J{是否医保?}
    J -- 是 --> K[医保出院结算]
    K --> L[医保报销计算]
    L --> M[个人自付计算]
    J -- 否 --> M
    M --> N[显示结算清单]
    N --> O[患者确认]
    O --> P[收取个人费用]
    P --> Q[打印结算发票]
    Q --> R[打印出院小结]
    R --> S[打印出院带药]
    S --> T[释放床位]
    T --> U[病案归档]
    U --> V([出院完成])
```

## 5. 检验管理流程

### 5.1 检验全流程

```mermaid
flowchart TD
    A([医生开具检验申请]) --> B[申请单发送至LIS]
    B --> C[生成标本条码]
    C --> D[护士采集标本]
    D --> D1[扫描条码-采集确认]
    D1 --> E[标本运送]
    E --> F[检验科接收标本]
    F --> G{标本合格?}
    G -- 否 --> H[拒收-通知重新采集]
    H --> D
    G -- 是 --> I[标本编号]
    I --> J[上机检测]
    J --> K[结果自动采集]
    K --> L{结果异常?}
    L -- 危急值 --> M[危急值报告]
    M --> N[通知临床科室]
    N --> O[临床确认接收]
    L -- 正常 --> P[结果审核]
    O --> P
    P --> Q{审核通过?}
    Q -- 是 --> R[结果发布]
    R --> S[报告生成]
    S --> T[报告发送至HIS]
    T --> U[医生查看结果]
    U --> V([检验完成])
    Q -- 否 --> W[复检/复查]
    W --> J
```

## 11. 患者主索引(EMPI)管理流程

```mermaid
    A([新患者登记]) --> B[录入基本信息]
    B --> C[EMPI匹配查询]
    C --> D{是否匹配到已有记录?}
    D -- 是 --> E[疑似重复记录列表]
    E --> F[人工确认]
    F --> G{确认为同一患者?}
    G -- 是 --> H[合并患者记录]
    G -- 否 --> I[新建独立记录]
    D -- 否 --> I
    H --> J[更新交叉索引]
    I --> J
    J --> K[分配全局唯一ID]
    K --> L[同步至各业务系统]
    L --> M([EMPI更新完成])
```
