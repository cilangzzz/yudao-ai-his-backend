# 处方管理 API 接口文档

> **模块**: 门诊医生工作站
> **Controller**: `OpPrescriptionController`
> **基础路径**: `/his/prescription`
> **版本**: v1.0
> **更新日期**: 2026-06-18

---

## 接口概览

| 接口 | 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|------|
| 开立处方 | POST | `/create` | `his:prescription:create` | 创建新处方 |
| 更新处方 | PUT | `/update` | `his:prescription:update` | 更新处方信息 |
| 删除处方 | DELETE | `/delete` | `his:prescription:delete` | 删除未审核的处方 |
| 获取处方详情 | GET | `/get` | `his:prescription:query` | 根据ID查询处方 |
| 根据处方编号查询 | GET | `/get-by-no` | `his:prescription:query` | 根据处方编号查询 |
| 获取处分页 | GET | `/page` | `his:prescription:query` | 分页查询处方列表 |
| 获取处方明细 | GET | `/items` | `his:prescription:query` | 查询处方药品明细 |
| 根据挂号ID查询 | GET | `/list-by-register` | `his:prescription:query` | 查询挂号关联的处方 |
| 根据患者ID查询 | GET | `/list-by-patient` | `his:prescription:query` | 查询患者的处方历史 |
| 审核通过 | POST | `/audit-pass` | `his:prescription:audit` | 药师审核通过 |
| 审核退回 | POST | `/audit-reject` | `his:prescription:audit` | 药师审核退回 |
| 调配处方 | POST | `/dispense` | `his:prescription:dispense` | 药师调配药品 |
| 发药 | POST | `/send` | `his:prescription:send` | 确认发药 |
| 批量发药 | POST | `/batch-send` | `his:prescription:send` | 批量确认发药 |
| 退药 | POST | `/return` | `his:prescription:return` | 处方退药处理 |
| 记录皮试结果 | POST | `/record-skin-test` | `his:prescription:skin-test` | 记录皮试结果 |
| 待审核处方列表 | GET | `/pending-audit-list` | `his:prescription:query` | 获取待审核处方 |
| 待调配处方列表 | GET | `/pending-dispense-list` | `his:prescription:query` | 获取待调配处方 |
| 待发药处方列表 | GET | `/pending-send-list` | `his:prescription:query` | 获取待发药处方 |

### 3. 审核处方（通过）

**请求**
- 方法: `POST`
- 路径: `/his/prescription/audit-pass`
- 权限: `his:prescription:audit`

**参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 处方ID |
| pharmacistId | Long | 是 | 药师ID |
| pharmacistName | String | 是 | 药师姓名 |
| remark | String | 否 | 审核备注 |

**响应**

```json
{
  "code": 0,
  "msg": "success",
  "data": true
}
```

## 处方状态流转

```
待审核(1) → 已审核(2) → 已调配(3) → 已发药(4)
    ↓           ↓           ↓
  已退回(6)   已作废(5)   已退药(7)
```

| 状态值 | 说明 | 可操作 |
|--------|------|--------|
| 1 | 待审核 | 审核、作废 |
| 2 | 已审核 | 调配 |
| 3 | 已调配 | 发药 |
| 4 | 已发药 | 退药 |
| 5 | 已作废 | 无 |
| 6 | 已退回 | 修改后重新提交 |
| 7 | 已退药 | 无 |

---

## 处方类型

| 值 | 说明 |
|----|------|
| 1 | 西药处方 |
| 2 | 中药处方 |
| 3 | 草药处方 |

## 错误码

| 错误码 | 说明 |
|--------|------|
| 1_015_003_000 | 处方不存在 |
| 1_015_003_001 | 处方已审核，无法修改 |
| 1_015_003_002 | 处方已发药，无法修改 |
| 1_015_003_003 | 处方明细不存在 |
| 1_015_003_004 | 药品不存在 |
| 1_015_003_005 | 药品库存不足 |
| 1_015_003_006 | 该药品需要皮试 |
| 1_015_003_007 | 皮试结果阳性，禁止使用 |

## 前端对接示例

### 医生开立处方

```javascript
const createPrescription = async (data) => {
  const res = await request({
    url: '/his/prescription/create',
    method: 'POST',
    data
  })
  return res.data
}
```

### 药师审核处方

```javascript
const auditPrescription = async (id, pharmacistId, pharmacistName, passed, reason) => {
  const url = passed ? '/his/prescription/audit-pass' : '/his/prescription/audit-reject'
  const res = await request({
    url,
    method: 'POST',
    params: passed 
      ? { id, pharmacistId, pharmacistName }
      : { id, pharmacistId, pharmacistName, reason }
  })
  return res.data
}
```

### 获取待发药处方列表

```javascript
const getPendingSendList = async (deptId) => {
  const res = await request({
    url: '/his/prescription/pending-send-list',
    method: 'GET',
    params: { deptId }
  })
  return res.data
}
```
