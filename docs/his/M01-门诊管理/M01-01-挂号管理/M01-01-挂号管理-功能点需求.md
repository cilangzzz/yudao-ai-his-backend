## 1. 功能概述

### 1.1 功能定位

挂号管理是门诊管理子系统的入口模块，负责患者挂号登记、号源管理、分诊排队、退号处理等核心功能。系统支持多种挂号方式（现场挂号、预约挂号、自助挂号、医保挂号、急诊挂号），实现号源的科学管理和合理分配。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短患者挂号等待时间 | 平均挂号时间≤2分钟 |
| 便捷目标 | 支持多渠道挂号方式 | 线上挂号占比≥50% |
| 管理目标 | 优化号源利用率 | 号源利用率≥85% |
| 体验目标 | 减少现场排队拥堵 | 候诊区排队人数≤20人 |

## 6. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OP-001 | 挂号患者身份校验 | 挂号时必须校验患者身份信息，支持身份证读卡器自动读取 |
| BR-OP-002 | 重复挂号校验 | 同一患者同一科室同一日不可重复挂号 |
| BR-OP-003 | 号源可用性校验 | 挂号时必须校验号源剩余数量 > 0 |
| BR-OP-004 | 医保身份实时验证 | 医保挂号必须实时验证医保资格 |
| BR-OP-005 | 预约挂号限次 | 每人每科室每日限约1次 |
| BR-OP-006 | 单日挂号次数上限 | 同一患者单日挂号次数不超过5次 |
| BR-OP-007 | 退号时间限制 | 已就诊不可退号 |
| BR-OP-008 | 预约超时处理 | 预约时间段结束后30分钟未签到自动过期 |
