## 1. 功能概述

### 1.1 功能定位

门诊收费管理是门诊管理子系统的结算模块，负责门诊患者费用的自动汇总、收费结算、医保结算、退费处理、发票管理等功能。系统支持多种支付方式，实现与医保系统的实时对接。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短收费等待时间 | 平均收费时间≤3分钟 |
| 准确目标 | 费用计算准确 | 费用差错率≤0.01% |
| 便捷目标 | 支持多种支付方式 | 支付方式覆盖率100% |
| 合规目标 | 医保结算规范 | 医保结算成功率≥99% |

### 1.3 功能范围

```

├── 费用管理
```

## 4. 接口设计

### 4.1 费用汇总接口

**接口路径**: `GET /api/op/charge/summary/{registerId}`

**响应结果**:
```json
{
  "code": 200,
  "data": {
    "registerId": 10001,
    "patientId": 1001,
    "items": [
      {"itemName": "专家挂号费", "price": 50.00, "qty": 1, "amount": 50.00},
      {"itemName": "阿莫西林胶囊", "price": 2.5, "qty": 24, "amount": 60.00}
    ],
    "totalAmount": 167.00
  }
}
```

### 4.2 收费结算接口

**接口路径**: `POST /api/op/charge/settle`

**请求参数**:
```json
{
  "registerId": 10001,
  "totalAmount": 167.00,
  "insurancePay": 60.40,
  "personalPay": 106.60,
  "payType": 2,
  "payDetails": [
    {"payType": "insurance", "amount": 60.40},
    {"payType": "wechat", "amount": 106.60}
  ]
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "收费成功",
  "data": {
    "chargeId": 20001,
    "invoiceNo": "FP202606170001",
    "totalAmount": 167.00,
    "payTime": "2026-06-17 09:35:00"
  }
}
```

### 4.3 退费接口

**接口路径**: `POST /api/op/charge/refund`

**请求参数**:
```json
{
  "chargeId": 20001,
  "refundItems": [
    {"itemId": 1, "refundQty": 1},
    {"itemId": 3, "refundQty": 24}
  ],
  "refundReason": "患者主动退费",
  "refundAmount": 110.00
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OP-009 | 收费身份核对 | 收费前必须核对患者身份 |
| BR-OP-010 | 医保实时结算 | 医保患者必须实时对接医保系统结算 |
| BR-OP-011 | 退费审批流程 | 退费必须有审批流程，部分退费需逐项确认 |
| BR-OP-012 | 日结对账 | 收费员每日必须日结对账 |
| BR-OP-013 | 发票唯一性 | 每笔收费对应唯一发票号 |
