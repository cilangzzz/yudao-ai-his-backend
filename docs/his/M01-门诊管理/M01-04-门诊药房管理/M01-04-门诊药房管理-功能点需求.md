## 1. 功能概述

### 1.1 功能定位

门诊药房管理是门诊管理子系统的药品发放模块，负责处方接收、处方审核、药品调配、发药确认、退药处理等功能。系统与药品管理子系统联动，实现药品库存的实时更新。

### 1.2 业务目标

| 目标类型 | 目标描述 | 衡量指标 |
|----------|----------|----------|
| 效率目标 | 缩短发药等待时间 | 平均发药时间≤5分钟 |
| 安全目标 | 处方审核覆盖 | 处方审核率100% |
| 准确目标 | 发药准确率 | 发药差错率≤0.01% |
| 管理目标 | 库存实时更新 | 库存更新及时率100% |

## 4. 接口设计

### 4.1 处方接收接口

**接口路径**: `GET /api/op/pharmacy/prescription/list`

**请求参数**:
```
status=1&date=2026-06-17
```

**响应结果**:
```json
{
  "code": 200,
  "data": [
    {
      "prescriptionId": 10001,
      "prescriptionNo": "CF202606170001",
      "patientName": "张三",
      "doctorName": "李主任",
      "totalAmount": 167.00,
      "status": 1,
      "createTime": "2026-06-17 09:30:00"
    }
  ]
}
```

### 4.2 发药确认接口

**接口路径**: `POST /api/op/pharmacy/dispense/{prescriptionId}`

**请求参数**:
```json
{
  "patientName": "张三",
  "identityVerified": true,
  "medicationGuidance": true,
  "pharmacistId": 1001
}
```

**响应结果**:
```json
{
  "code": 200,
  "msg": "发药成功",
  "data": {
    "prescriptionId": 10001,
    "dispenseTime": "2026-06-17 09:45:00"
  }
}
```

## 5. 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| BR-OP-013 | 发药身份核对 | 发药前必须核对患者身份（姓名+就诊号） |
| BR-OP-014 | 麻醉药品五专管理 | 麻醉药品、精神药品必须专项管理 |
| BR-OP-015 | 库存预警 | 药品库存低于安全库存时自动预警 |
| BR-OP-016 | 效期预警 | 效期≤3个月药品自动预警，≤1个月禁止调配 |
