## 1. 项目概述

### 1.1 项目背景

基于 yudao-vue-pro（芋道源码）框架开发 HIS 智慧医疗信息系统，目标达到 HIMSS EMRAM Stage 5+ 水平。

### 1.2 核心目标

| 目标类型 | 具体目标 | 量化指标 |
|----------|----------|----------|
| 业务目标 | 支撑医院日常核心业务运转 | 日均5000+门诊量、1000+住院量 |
| 质量目标 | 满足HIMSS EMRAM Stage 5+标准 | 闭环给药核对率100%、危急值15分钟通报 |
| 安全目标 | 满足等保三级安全要求 | 审计日志完整、数据加密存储 |
| 技术目标 | 支持微服务架构演进 | 模块独立部署、接口标准规范 |

### 1.3 技术栈

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 后端框架 | Spring Boot | 3.2.x |
| 前端框架 | Vue 3 + Element Plus | 3.4.x |
| 数据库 | MySQL | 8.0.x |
| 缓存 | Redis | 7.x |
| 消息队列 | RabbitMQ | 3.12.x |
| 对象存储 | MinIO | Latest |

## 4. Agent 任务分配

### 4.1 Agent 定义

| Agent 名称 | 负责模块 | 主要职责 |
|------------|----------|----------|
| system-agent | M09 | 用户、角色、权限、字典、科室管理 |
| outpatient-agent | M01 | 挂号、门诊工作站、收费、药房 |
| inpatient-agent | M02 | 入院、医嘱、护理、eMAR、出院 |
| pharmacy-agent | M06 | 药库、药房、处方审核、CDS |
| emr-agent | M03 | 病历模板、编辑、审签、归档 |
| lis-agent | M04 | 检验申请、标本、结果、报告 |
| ris-agent | M05 | 影像申请、检查、存储、报告 |
| finance-agent | M08 | 收费项目、医保结算、费用记账 |
| integration-agent | M10 | EMPI、主数据、消息引擎 |
| surgery-agent | M07 | 手术排期、麻醉记录 |
| patient-agent | M11 | 患者门户、预约、报告查询 |
| operation-agent | M12 | 运营看板、统计报表 |
| ai-agent | M13 | AI辅助诊断 |

### 4.2 Agent 执行流程

```
1. system-agent 先行执行 (M09 系统管理)
   ↓
2. outpatient-agent 并行执行 (M01 门诊管理)
   pharmacy-agent 并行执行 (M06 药品管理)
   ↓
3. inpatient-agent 执行 (M02 住院管理)
   ↓
4. emr-agent 并行执行 (M03 电子病历)
   lis-agent 并行执行 (M04 检验管理)
   ris-agent 并行执行 (M05 影像管理)
   finance-agent 并行执行 (M08 财务管理)
   ↓
5. integration-agent 执行 (M10 集成平台)
   ↓
6. surgery-agent 并行执行 (M07 手术麻醉)
   patient-agent 并行执行 (M11 患者服务)
   operation-agent 并行执行 (M12 运营管理)
   ai-agent 并行执行 (M13 AI辅助)
```
